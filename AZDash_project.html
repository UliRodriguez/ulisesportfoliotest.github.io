WorkfLow Health Dashboard

In this project I developed a Tableau dashboard to help monitor LinkedIn's internal tool that runs workflow jobs.

I built a dashboard in Tableau to help monitor workflow jobs from an internal scheduler tool. My manager and I constructed a SQL code to extract information about projects that our team owned from two datasets and combine them to create one table. I then plugged the data table into Tableau and created three view levels: summary for a single flow, summary stats of multiple flows over the last month, and detailed view.

In the single flow view a user could select a flow and view its last successful run date and success rate to see the relevance and consistency of the flow job. In the multiple flows over the last month view, the user can see additional information like the average runtime of each flow that my team owns and the total hours that the jobs ran for each day. The detailed view is a table that contains all the relevant data of each job flow like the name, start time, end time, duration, the cluster that the flow lives in, and the user who owns the job. Each view level can be filtered by project name and flow name or consistency and easy tracking.

The issue my team wanted me to address was not having a centralized place to monitor my team's workflow jobs and view when jobs ran successfully or failed. This dash would help keep track of how the scheduler tool is working in general and take immediate action if something would to go wrong. This would be a great place to find information about workflow jobs owned by the team, for example, the average runtime of jobs, the success rate of said job, and a direct link to the execution log if the job failed.
